# Resume Page Constitution

## Core Principles

### I. Single-Page Simplicity
Every feature for this project must support a single static page experience with clear sections and no unnecessary navigation or multi-page complexity.

### II. Resume-First Content Structure
The page must present the person’s resume information in a logical, scannable order, including sections for Summary, Skills, Years of Experience and Work History, and Technologies Worked On.

### III. Maintainable Content
Resume content should be easy to update and maintain. Information should be organized in a simple, structured way so changes do not require redesigning the entire page.

### IV. Readability and Accessibility
The page must use semantic HTML, readable typography, strong contrast, responsive layout, and accessible interactions so the content is easy to read on different devices.

### V. Lightweight Delivery
The implementation should remain fast, dependency-light, and reliable, favoring simple static assets over heavy frameworks or backend requirements.

## Additional Constraints

The page must be built as a single static resume page that clearly displays the following sections:
- Summary
- Skills
- Years of Experience and Work History
- Technologies Worked On

Content should be presented in a polished, professional layout that highlights the candidate’s experience without unnecessary visual clutter.

## Development Workflow

Changes to the resume content or layout should be made with clarity and consistency in mind. New sections or visual updates must preserve the single-page structure and keep the content easy to maintain.

## Governance

This constitution supersedes ad-hoc layout or content decisions for the resume page. Any major change to structure, section ordering, or styling should be documented and reviewed before implementation.
